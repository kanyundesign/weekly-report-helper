# 📋 Design Task 任务卡片管理规范

> 为了让周报系统能够准确识别和生成周报，请按照以下规范管理你的任务卡片。

---

## 1️⃣ 任务基础信息

### 任务标题
- ✅ 简洁明了，描述核心内容
- ✅ 避免过长，建议 15 字以内
- ❌ 不要包含日期、状态等冗余信息

**好的示例：**
```
集团工牌需求设计
```

**不好的示例：**
```
【进行中】12月集团工牌需求设计-yifan负责
```

### 负责人 (Assignee)
- 必须选择自己作为负责人
- 周报系统根据负责人筛选任务

### 任务状态 (Status)

| 状态 | 说明 | 周报归类 |
|------|------|----------|
| `Next Up` | 计划中，尚未开始 | 本周计划 |
| `In Progress` | 正在进行中 | 本周计划 |
| `Review` | 等待评审/验收 | 本周计划 |
| `Done` | 已完成 | 上周完成（7天内） |

---

## 2️⃣ 排期时间（重要⚠️）

### 必须填写
- **开始日期**：任务开始时间
- **结束日期**：预计完成时间

### 格式要求
- 使用 Notion 的日期属性
- 周报系统根据时间范围判断任务归属

---

## 3️⃣ 子任务拆分规范（核心⭐）

### 工时单位

| 单位 | 含义 |
|------|------|
| `1pd` | 1人1天的工作量（约8小时） |
| `0.5pd` | 半天工作量（约4小时） |
| `2pd` | 2天工作量 |
| `1h` / `2h` | 按小时计算（适合小任务） |

### 子任务格式

在任务卡片内容中，按以下格式书写子任务：

```
1. 需求分析与调研 1pd
2. 方案设计 2pd
3. 视觉设计 1.5pd
4. 设计评审 0.5pd
5. 设计修改与交付 1pd
```

### 标记完成

已完成的子任务在前面加 `✅` 或使用 Notion 的 To-do 格式：

```
✅ 1. 需求分析与调研 1pd
✅ 2. 方案设计 2pd
3. 视觉设计 1.5pd（进行中）
4. 设计评审 0.5pd
5. 设计修改与交付 1pd
```

**进度计算示例**：
- 已完成工时：1pd + 2pd = 3pd
- 总工时：1pd + 2pd + 1.5pd + 0.5pd + 1pd = 6pd
- 进度：3pd / 6pd = **50%**

## 4️⃣ 卡片内容规范

### ✅ 推荐的写法

```
【任务背景】
简述需求来源和目标

【子任务】
1. 需求分析 0.5pd
2. 方案设计 1pd
3. 视觉输出 1.5pd

【相关文档】
→ 点击查看设计稿（链接到子页面）
```

### ❌ 避免的写法

- 直接粘贴大段文字、会议记录
- 嵌入过多图片（影响读取速度）
- 复杂的表格和数据库

### 📄 大量内容的处理方式

如果需要管理以下内容：
- 详细的需求文档
- 大段的会议记录
- 多个设计稿链接
- 版本迭代记录

**请在卡片内新建子页面（Page）来管理**，保持主卡片内容简洁。

操作方式：在卡片内容中输入 `/page`，创建一个子页面来存放详细内容。

---

## 5️⃣ 最佳实践

### 每日更新
- 及时更新任务状态
- 完成子任务后标记 ✅
- 调整排期时同步更新日期

### 任务粒度
- 单个任务建议 1-5 天完成
- 超过 5 天的大任务应拆分为多个子任务
- 每个子任务最好不超过 2pd

### 周一前检查
- 确认上周完成的任务状态为 `Done`
- 确认本周计划的任务状态正确
- 检查子任务完成标记是否准确

---

## 6️⃣ 常见问题

### Q: 为什么我的任务没有出现在周报中？

检查以下几点：
1. Assignee 是否选择了自己
2. 任务状态是否正确（Next Up / In Progress / Review / Done）
3. 任务日期是否在本周范围内

### Q: 为什么进度计算不准确？

确保：
1. 子任务格式正确：`任务描述 + 空格 + 工时`
2. 已完成的子任务有 ✅ 标记
3. 工时格式正确：`1pd`、`0.5pd`、`2h` 等

### Q: 临时任务怎么处理？

1. 创建任务卡片，设置当天日期
2. 添加子任务并标记完成
3. 完成后及时改为 `Done` 状态

### Q: 跨周的任务怎么处理？

- 任务会根据当前状态和编辑时间自动归类
- 如果任务跨周，会显示当前的完成进度
- 完成后改为 `Done`，会出现在"上周完成"中

---

## 📌 提交周报前的快速检查清单

每周提交周报前，请检查以下项目：

- [ ] 所有任务都有正确的负责人
- [ ] 所有任务都有开始/结束日期
- [ ] 任务状态与实际进度一致
- [ ] 子任务有明确的工时标注（如 1pd、0.5pd）
- [ ] 已完成的子任务有 ✅ 标记
- [ ] 大量内容已移至子页面管理
